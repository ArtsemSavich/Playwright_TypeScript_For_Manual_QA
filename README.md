# Playwright for Manual QA Engineers — 6-Week Learning Path

An interactive, self-paced course that takes manual QA engineers from zero to writing real Playwright tests with TypeScript. No prior coding experience required.

🔗 **[Open Course →](https://artsem savich.github.io/Playwright_TypeScript_For_Manual_QA/index.html)**  
*(or clone the repo and open `index.html` in your browser)*

---

## What is this?

A single-file interactive HTML course designed specifically for **manual QA engineers** who want to learn test automation with Playwright + TypeScript.

The course is built for people who:
- Have QA experience but no (or minimal) coding background
- Want to learn Playwright from scratch in a structured way
- Prefer hands-on practice over just reading theory

---

## Course Structure

| Week | Topic | Time |
|------|-------|------|
| **Week 0** | Setup — Terminal, Node.js, VS Code, Git | Pre-work |
| **Week 1** | Your First Playwright Project & Test | 5–6 hours |
| **Week 2** | TypeScript for QA | 8–10 hours |
| **Week 3** | Real UI Testing with Playwright | 7–8 hours |
| **Week 4** | Debugging & Test Structure | 6–7 hours |
| **Week 5** | Git & Final Project Setup | 5–6 hours |
| **Week 6** | Final Project — Page Object Model & Test Suite | 6–7 hours |

**Total:** ~40–50 hours of structured learning

---

## What You'll Learn

- Setting up a Playwright project from scratch
- Writing tests with TypeScript (`async/await`, types, interfaces)
- Working with locators: `getByRole`, `getByTestId`, `getByText`, CSS, XPath
- Validating locators with DevTools (`$$('selector')`)
- Understanding auto-waiting and actionability
- Debugging tests with Playwright trace viewer
- Organizing tests with `describe`, `beforeEach`, fixtures
- Git basics for QA: commits, branches, pull requests
- Page Object Model (POM) for maintainable test suites

---

## How to Use

1. Clone or download this repository
2. Open `index.html` in your browser
3. Start from **Week 0** (setup) and work through each week in order
4. Use the built-in checkboxes to track your progress
5. Each week includes: theory → exercises → "ready when" checklist

> The course uses [SauceDemo](https://www.saucedemo.com) as the main test target — a real, publicly available demo app.

---

## Features

- **Progress tracking** — checkboxes saved in `localStorage` (persist between sessions)
- **Interactive exercises** — practical tasks with hints and expected assertions
- **Cheat sheets** — actions, assertions, locator strategy reference
- **No dependencies** — single HTML file, works offline, no server needed
